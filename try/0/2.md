# 試しにやってみる

以下のような原稿の構文解析器を作りたい。開発環境Bun、言語TypeScript。

Japan Novel = Javel。日本語小説構文解析器。Javel Parser。原稿を先頭から1字ずつ読み込んでトークンに分割し、構文木のノードを生成する。最終的に構文木を返却する。

十万字規模の原稿にも耐えられるようトップレベルのAstNodeを一個ずつ返却する。

実装は単一責任の原則を死守する。責任分界点を徹底的に細分化する。

```javel
題名
著者名

＊＊＊

完

読了ありがとうございました。

＃見出し

段落。
形式段落。

二つ目の段落。

＊＊＊

主題区切り。

＃＃見出し２

日本語用インライン要素。

あ漢字《かんじ》。
超｜漢字《かんじ》超｜KANJI《かんじ》。

；傍点；〔割注〕。

＃＃＃見出し３

｛エスケープ。漢字《かんじ》；傍点；〔割注〕。｝

｛　｛｝　｝

｛　｛漢字《かんじ》｝　｝のように記述すればエスケープできます。

｛＃＃＃見出しじゃない｝

｛＊＊＊｝

主題区切りじゃない。

＃＃＃｛＃｝見出し４じゃなく３

＃＃＃＃見出し４

＃＃＃＃＃見出し５

＃＃＃＃＃＃見出し６
```


* 要素
	* ブロック
		* メタ文字
			* 改行
				* 改行二連続以上でブロック要素区切りとする
	* インライン
		* メタ文字
			* 改行
				* 改行二連続以上でブロック要素区切りとする
