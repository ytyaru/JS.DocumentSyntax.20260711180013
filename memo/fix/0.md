# Hudolap: 人間可読文書化言語兼構文解析器生成器

* Hudolap: Human readable documentation language and parser generator (人間可読文書化言語兼構文解析器生成器)
	* File (物理バイナリの文脈：OS・ファイル形式の定義)
		* Validator / Normalizer
	* Text (文字列・メモリの文脈：サロゲート、結合文字等の防衛)
		* Validator / Normalizer
	* Data (構造化データの文脈：リスト、KeyValue、行列、グラフの基本型)
	* Affix (接辞の文脈：YAML風味のインデント・代入・複数行テキスト)
	* Document (文書の空間表現の文脈：最も主観的・ヒューリスティックな表現)
		* Structure (文書を構成する4つの絶対的な物理の檻：Fence/Block/Blank/Inline)
		* Parser (4つの檻を維持して上から順に1ノードずつ yield するピュアな走査インフラ)
			* Tokenizer (Tokenを生成)
			* Lexer (AstNodeを生成)
		* Builder (文法部品を投入し、メタ文字の衝突を検証して、1つの安全なLanguageを組み立てる工場)
			* Language (【無形の法則の集合】Parserが解析できる完成されたルール群。JavelLanguage等)
				* Grammar (独立した1つの文法解析アルゴリズム。例: ルビ文法)
					* GrammarPattern (Hudolap標準の抽象アルゴリズムの型)
				* Syntax (Morphemeに解体する個別の構文規則)
					* Morpheme (親文字やメタ文字など、原稿復元用の最小要素)
	* Helper
		* LanguageBuilder
			* GrammarBuilder
				* SyntaxBuilder
					* MorphemeBuilder
* Hudoc: Hudlapを使用して作成された独自体系
	* Javel
	* 独自Wiki記法
	* Markdown類似記法
	* ...

```

接頭辞
---
内容

接尾辞
===
```

```
===
```

##### Grammar

###### GrammarParser

* Fence
	* `===型識別子\n接頭辞\n---\n\n内容\n\n---接尾辞\n===`（引数で多様なパターンに可能。単行・複数行、接辞箇所組合せ7パターン、内容の任意・必須等超複雑）
* Blank
	* `\n\n\n...`
* Block
	* AffixedBlock: 接辞にメタ文字が付与されるパターン(接辞箇所7パターン)
	* AllHeadBlock: 全行行頭にメタ文字が付与されるパターン
	* IndentBlock: 字下げパターン（先頭行以外にメタ文字（インデント）が付与されうるパターン。AffixedBlockで実装可？）
	* OnlyBlock: メタ文字のみで構成されたパターン（複数行に跨る可能性もあり？　AffixedBlockで実装可？）
* Inline
	* VariableLengthEnclosed（可変長挟み込み型。`傍点（；傍点；）、割注（〔割注〕）、入れ子エスケープ（；；　；傍点；　；；）、｛｛｛｝｝｝`）
	* ThreePartInline（順方向3パーツ型。`｜漢字《かんじ》`）
	* ReverseScanInline（逆方向解析型。基点特定パターンをGrammarPatternから作りだし、始点特定は文字種または指定字数のどちらかの2つのタイプで引数を渡し指定する。`漢字《かんじ》`）
	* MultiEnclosedInline（複数キー連動挟み込み型。`[文字](URL)、[![ALT](a.png)](t.html)`）
